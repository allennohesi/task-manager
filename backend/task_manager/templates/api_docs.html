Complete API Documentation

Base URL: http://localhost:8000

ðŸ“– Overview

The Task Manager API is a RESTful API for managing tasks. It supports creating, reading, updating, and soft-deleting tasks. All endpoints return JSON data.

Note: Tasks with status 1 or 2 are considered deleted and are excluded from list and retrieve operations.

ðŸ”— Endpoints

GET /api/tasks/
List all active tasks - Returns a list of all tasks that are not deleted (status != 1 and status != 2).
Response Example:
[ { "id": 1, "title": "Complete project", "description": "Finish the task manager project", "completed": false, "status": 0, "created_at": "2025-11-29T20:00:00Z", "updated_at": "2025-11-29T20:00:00Z", "is_deleted": false } ]
200 OK
POST /api/tasks/
Create a new task - Creates a new task with the provided data.
Request Body:
{ "title": "New Task", "description": "Task description (optional)", "completed": false }
Response Example:
{ "id": 2, "title": "New Task", "description": "Task description", "completed": false, "status": 0, "created_at": "2025-11-29T20:05:00Z", "updated_at": "2025-11-29T20:05:00Z", "is_deleted": false }
201 Created 400 Bad Request
GET /api/tasks/{id}/
Retrieve a specific task - Returns the details of a single task by its ID.
Response Example:
{ "id": 1, "title": "Complete project", "description": "Finish the task manager project", "completed": false, "status": 0, "created_at": "2025-11-29T20:00:00Z", "updated_at": "2025-11-29T20:00:00Z", "is_deleted": false }
200 OK 404 Not Found
POST /api/tasks/{id}/
Update a task (full update) - Updates all fields of a task. All fields must be provided.
Request Body:
{ "title": "Updated Task", "description": "Updated description", "completed": true }
200 OK 400 Bad Request 404 Not Found
PATCH /api/tasks/{id}/
Partially update a task - Updates only the provided fields. Other fields remain unchanged.
Request Body:
{ "completed": true }
200 OK 400 Bad Request 404 Not Found
DELETE /api/tasks/{id}/
Soft delete a task - Sets the task status to 1 (deleted). The task is not permanently removed from the database.
204 No Content 404 Not Found
POST /api/tasks/{id}/mark_completed/
Mark a task as completed - Sets the completed field to true for the specified task.
Response Example:
{ "id": 1, "title": "Complete project", "description": "Finish the task manager project", "completed": true, "status": 0, "created_at": "2025-11-29T20:00:00Z", "updated_at": "2025-11-29T20:10:00Z", "is_deleted": false }
200 OK 404 Not Found
POST /api/tasks/{id}/mark_incomplete/
Mark a task as incomplete - Sets the completed field to false for the specified task.
Response Example:
{ "id": 1, "title": "Complete project", "description": "Finish the task manager project", "completed": false, "status": 0, "created_at": "2025-11-29T20:00:00Z", "updated_at": "2025-11-29T20:15:00Z", "is_deleted": false }
200 OK 404 Not Found

ðŸ“Š Task Model

Task Fields:
  • id (integer, read-only) - Unique identifier
  • title (string, required) - Task title (max 200 characters)
  • description (string, optional) - Task description
  • completed (boolean, default: false) - Completion status
  • status (integer, default: 0) - Task status (0: Active, 1-2: Deleted)
  • created_at (datetime, read-only) - Creation timestamp
  • updated_at (datetime, read-only) - Last update timestamp
  • is_deleted (boolean, read-only) - Computed property indicating if task is deleted
